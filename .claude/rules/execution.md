# Execution

## When executing a task

- Always follow the plan in the task file
- Always start by writting the suite of tests that match the acceptance criteria
- Always end a change request by running the tests and ensuring they pass
- Tests are allowed to fail when the code is not yet written or is partially written
- Always update the task file with the lessons learned
- Always update the task file with the next steps
- Always update the task file with the improvements
- Always document potential technical debt and risks added by the changes
