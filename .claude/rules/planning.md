# Planning

## When planning a new feature

- Always start a new task, creating a file under the `~/.claude/plans/tasks/` directory
- Always start with a detailed plan in a markdown file
- Include the feature name, link references, description, acceptance criteria, and any relevant details
- Include the expected impact on the codebase
- Include the expected timeline for the feature
- Include the expected resources required for the feature
- Include the expected risks and mitigations for the feature
- Include the expected dependencies for the feature
- Include the expected test cases for the feature
- Include the expected documentation for the feature
